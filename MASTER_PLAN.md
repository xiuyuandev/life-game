# LifeUp（人生升级）— 全局架构与产品定位文档

> 本文档基于产品方向、技术选型、技能系统、数值策划、数据同步等核心决策的综合整理。它是后续所有开发工作的基准。

---

## 一、核心产品定位

### 1.1 我们不是什么

LifeUp **不是一款游戏**。它不承担娱乐消遣的使命，不需要"杀时间"，不追求让用户无意义地沉迷在屏幕里。

### 1.2 我们是什么

LifeUp 是一个**用游戏化外壳承载真实人生成长的数据化工具**。

核心使命只有一句话：

> **让玩家通过真实地投入时间学习技能，感受到自己在现实生活中变得越来越强大、越来越幸福。**

### 1.3 核心用户价值

| 用户痛点 | LifeUp 的解法 |
|---------|--------------|
| 不知道时间花哪了 | 柳比歇夫式精确时间记录 + 日/周/月/年报 |
| 学了东西没成果感 | 技能等级可视化 + 累计投入时间档案 |
| 难以坚持 | streak（连续天数）+ 熟练度徽章 + 展厅收集欲 |
| 看不到技能之间的联系 | 技能组合系统，发现复合能力 |
| 焦虑、觉得没进步 | 每天2-3小时即可看到数据上的成长，降低心理门槛 |

### 1.4 玩家心理模型：收集 + 养成 + 生产

```
现实中花时间学习 → 游戏里技能升级 → 获得能力标签 → 用于实际生产/生活
     ↑                                                              ↓
     └──────────── 看到成果后更有动力投入 ──────────────────────────┘
```

**关键认知：** 技能的价值最终要落回现实。游戏里的等级只是**现实世界能力的镜像和量化**。

### 1.5 技能示例与组合逻辑

**单一技能的现实映射：**
- 编程 → 找到开发工作、做副业、自动化生活
- 英语 → 看原版资料、出国交流、外企机会
- 烹饪 → 健康饮食、生活幸福感、社交货币

**组合技能（Combo Skills）：**
当两个相关技能都达到一定等级时，解锁"组合技能"：
- 英语 Lv.3 + 恋爱沟通 Lv.3 → 解锁「跨文化社交」
- 编程 Lv.3 + 英语 Lv.3 → 解锁「远程工作/出海开发」
- 烹饪 Lv.2 + 摄影 Lv.2 → 解锁「美食自媒体」

## 二、技术架构方案

### 2.1 技术栈：Kotlin + Jetpack Compose（纯原生 Android）

**选型决策：纯原生 Android 开发**

| 维度 | 选择理由 |
|-----|---------|
| 原生体验 | 100% Android 原生渲染，无 WebView 套壳，流畅度最高 |
| 性能 | 直接调用系统 API，启动速度和运行效率最优 |
| 体积 | APK 体积小，无框架运行时开销 |
| Material You | 完美支持 Android 12+ 动态取色和 Material 3 设计语言 |
| 生态 | Google 官方推荐方案，文档和社区支持最完善 |
| 长期维护 | Kotlin 是 Android 官方语言，Jetpack Compose 是官方 UI 框架 |

### 2.2 核心技术组件

| 组件 | 用途 | 版本 |
|-----|------|------|
| **Kotlin** | 主开发语言 | 2.0+ |
| **Jetpack Compose** | 声明式 UI 框架 | BOM 2024+ |
| **Material 3** | 设计系统 + 动态取色 | 1.3+ |
| **Room** | 本地 SQLite ORM | 2.6+ |
| **Hilt** | 依赖注入 | 2.51+ |
| **Kotlin Coroutines + Flow** | 异步与响应式数据流 | 1.8+ |
| **Navigation Compose** | 页面路由 | 2.7+ |
| **DataStore** | 轻量键值存储（设置项） | 1.1+ |
| **WorkManager** | 后台定时任务（每日备份） | 2.9+ |

## 三、数据存储与云端备份架构

### 3.1 核心原则

> **所有数据以本地为绝对中心。云端唯一的作用是「异地备份」，不存在本地与云端的冲突概念。**

### 3.2 本地存储方案

- **主数据库**：Room（SQLite ORM），Android 原生方案
- **设置项**：DataStore（键值对），替代 SharedPreferences
- **本地备份目录**：应用私有目录下的 `backups/` 文件夹
- **备份策略**：保留最近 30 天的本地每日快照

### 3.3 云端备份方案：双方案可切换

#### 方案 A：S3 兼容存储（七牛云 / AWS S3 / 阿里云 OSS）

| 项目 | 说明 |
|-----|------|
| 存储内容 | AES-256 加密后的完整数据快照 |
| 认证方式 | AccessKey + SecretKey + Bucket + Endpoint |
| 文件命名 | `lifeup_backup_<device_id>_<yyyy-MM-dd>.enc` |

#### 方案 B：WebDAV（坚果云 / 群晖 / Nextcloud）

| 项目 | 说明 |
|-----|------|
| 存储内容 | 同上 |
| 认证方式 | URL + 用户名 + 密码 |
| 文件路径 | `/LifeUp/backups/lifeup_backup_<device_id>_<yyyy-MM-dd>.enc` |

### 3.4 云端备份策略：按天保留，自动轮转

- 每日应用启动时自动检测并推送备份
- 云端始终保留 **最近 10 天** 的备份
- App 永远不会主动从云端拉取数据
- 用户可手动推送或手动拉取恢复

### 3.5 数据恢复流程

设置 → 数据恢复 → 展示云端10个历史备份 → 选择 → 预览摘要 → 确认覆盖（恢复前自动保存本地快照）

## 四、技能系统核心设计

### 4.1 等级体系：五阶人生

| 阶位 | 命名 | 现实对应 | 感知设计 |
|-----|------|---------|---------|
| **一阶** | **启程** | 刚刚入门 | 新技能默认，获得「启程者」徽标 |
| **二阶** | **笃行** | 能坚持练习，产出稳定 | 累计 40h + streak 7天，卡片铜色边框 |
| **三阶** | **专精** | 熟练掌握，可以此谋生 | 累计 160h + streak 30天，卡片银色边框 |
| **四阶** | **化境** | 融会贯通，能创新 | 累计 480h + streak 60天，卡片金色边框+微光 |
| **五阶** | **传世** | 顶尖水平，能教导他人 | 累计 1200h + streak 180天，传说级特效+C位置顶 |

**阶位门槛可自定义：** 用户可为每个技能单独设置门槛，覆盖默认值。

### 4.2 技能分类体系

| 大类 | 主题色 | 示例 |
|-----|--------|------|
| 谋生技能 | 科技青 | 编程、设计、翻译 |
| 社交技能 | 暖橙 | 恋爱沟通、幽默表达 |
| 语言技能 | 宝石蓝 | 英语、日语 |
| 生活技能 | 草木绿 | 烹饪、理财 |
| 身体技能 | 活力红 | 跑步、力量训练 |
| 心智技能 | 紫罗兰 | 批判性思维、情绪管理 |
| 复合技能 | — | 由两个基础技能解锁 |

### 4.3 技能组合（Combo）系统

用户可自定义组合规则，每条规则包含：规则名称、主/副技能、阶位要求、经验加成、现实建议、连接线颜色。

当两个技能同时满足条件时自动激活，提供经验加成和现实行动建议。

### 4.4 展览馆式展示

```
技能展览馆
    ├── 展厅「2025年度目标」（用户自定义）
    │       ├── 编程 · 专精
    │       └── 英语 · 笃行
    ├── 展厅「谋生技能馆」
    │       ├── 编程
    │       └── 设计
    └── 未归档技能
```

每个展厅有自定义名称、主题色、图标，内部可拖拽排序，有封面技能（C位）。

### 4.5 技能图标系统

| 层级 | 名称 | 解锁条件 | 风格 |
|-----|------|---------|------|
| 基础库 | 轮廓图标 | 默认可用 | 2px 线框风格 |
| 进阶库 | 面性图标 | 笃行解锁 | 单色填充+微渐变 |
| 传说库 | 动态徽标 | 传世解锁 | 带微动效 |

支持用户上传自定义 SVG（专精后解锁）。

- 应用设置 + 数据导出

## 六、数值策划体系

### 6.1 核心数值哲学

> **时间是最稀缺的资源。数值系统的使命是让每一分钟都被看见、被肯定、被积累。**

### 6.2 核心资源三角：能量 · 经验 · 金币

| 资源 | 获取方式 | 主要消耗 | 设计意图 |
|-----|---------|---------|---------|
| **能量** | 完成今日待办 | 创建技能、启动计时、补录时间 | 限制操作频率，鼓励先规划再行动 |
| **经验** | 投资性计时 | 无消耗，自动累积 | 量化时间投入的成长价值 |
| **金币** | 所有计时 | 购买装备、解锁外观 | 让所有时间都有经济回报 |

### 6.3 能量值系统

| 行为 | 能量变化 | 限制 |
|-----|---------|------|
| 完成待办 1 项 | +1 点 | 每天最多 10 项 |
| 创建新技能 | -2 点 | — |
| 启动技能计时 | -1 点 | — |
| 手动补录时间 | -1 点 | — |

每日 00:00 重置为 0，不累积。

### 6.4 经验值获取体系（乘区制）

```
基础经验 = 投入分钟数 × 1 EXP/分钟

总经验 = 基础经验 × 阶位乘区 × 装备乘区 × 组合乘区 × Streak乘区 × 首击乘区 × 衰减系数
```

| 乘区 | 计算方式 |
|-----|---------|
| 阶位乘区 | 启程1.0x / 笃行1.12x / 专精1.28x / 化境1.48x / 传世1.75x |
| 装备乘区 | 所有装备效果加算后转换 |
| 组合乘区 | 激活组合中最高加成，默认1.10x |
| Streak乘区 | 1 + min(连续天数,30) × 0.005，上限1.15x |
| 首击乘区 | 每日首次投资性计时 1.20x |
| 衰减系数 | 见下方 Soft Cap |

### 6.5 每日收益 Soft Cap

| 当日累计投入 | 衰减系数 |
|------------|---------|
| 0 - 60 分钟 | 100% |
| 60 - 120 分钟 | 90% |
| 120 - 180 分钟 | 72% |
| 180 - 240 分钟 | 50% |
| 240 - 300 分钟 | 30% |
| 300 分钟+ | 15% |

### 6.6 阶位晋升数值

| 阶位 | 累计时间 | Streak | 每天2h预计 |
|-----|---------|--------|-----------|
| 启程 | 0h | 0天 | 立即 |
| 笃行 | 40h | 7天 | 约3周 |
| 专精 | 160h | 30天 | 约2.5个月 |
| 化境 | 480h | 60天 | 约8个月 |
| 传世 | 1200h | 180天 | 约2年 |

### 6.7 金币经济系统

**获取：**
| 活动类型 | 金币/分钟 |
|---------|----------|
| 投资性计时 | 1 金币 |
| 消耗性计时 | 0.2 金币 |
| 每日首击奖励 | +50 金币 |
| 阶位晋升 | 笃行+500 / 专精+2000 / 化境+5000 / 传世+10000 |

**用途：** 购买装备(500-5000G) / 解锁图标库 / 角色外观 / 重置技能

### 6.8 角色属性计算

六维属性由技能自动计算：

```
力量 = Σ(身体技能阶位值) × 2 + Σ(其他) × 0.2
智力 = Σ(谋生+语言+心智阶位值) × 2 + Σ(其他) × 0.2
魅力 = Σ(社交+生活阶位值) × 2 + Σ(其他) × 0.2
耐力 = 最高streak × 0.5 + 技能总数 × 1
敏捷 = 本周投资性活动类别数 × 3 + 组合激活数 × 2
运气 = 随机1~5 + 传世技能数 × 2
```

### 6.9 柳比歇夫时间记录法

两种记录方式：
- **实时计时**：点击开始，实时记录
- **手动补录**：输入起始和截止时间（精确到分钟），系统自动计算时长

单条记录不超过12小时，重叠时间段提示确认。

### 6.10 健康与反内卷设计

| 机制 | 说明 |
|-----|------|
| 平滑收益衰减 | 同一技能当日超1小时后收益平滑下降 |
| 健康提醒 | 连续计时超90分钟弹出休息提醒 |
| Streak 睡眠保护 | 23:00-06:00 计时不计入 streak |
| 消耗性时间温和记录 | 刷手机也有微量回报，不制造焦虑 |
| 每周休息日提示 | 连续7天有记录时提示可休息 |

## 七、视觉与交互设计

### 7.1 视觉定位

> **精致的数据化人生仪表盘**

- **风格**：Material 3 + 游戏化数据可视化
- **色彩**：深色模式为主，技能卡片用自定义色点缀，支持 Android 12+ 动态取色
- **动效**：升级庆祝动效，日常界面克制
- **字体**：系统默认 + 等宽数字字体展示时间数据

### 7.2 交互原则

| 原则 | 实践 |
|-----|------|
| 即时反馈 | 计时结束1秒内弹出结算面板 |

## 十、总结

LifeUp 的终极愿景是成为用户的**"第二大脑"和"人生操作系统"**。

它不替代真实生活，而是**放大真实生活的意义**。当用户在现实中花2小时学英语，打开 LifeUp 看到自己的「英语」技能从启程走向笃行，看到「累计投入127小时」的档案，看到「跨文化社交」的组合技能被点亮——这种**确定性、可量化、可积累**的感觉，就是我们想给用户的全部。

**时间没有浪费。每一分钟都在让你升级。**
