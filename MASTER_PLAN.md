# LifeUp（人生升级）— 全局架构与产品定位文档

> 本文档基于产品方向、技术选型、技能系统、数值策划、数据同步等核心决策的综合整理。它是后续所有开发工作的基准。

---

## 一、核心产品定位

### 1.1 我们不是什么

LifeUp **不是一款游戏**。它不承担娱乐消遣的使命，不需要"杀时间"，不追求让用户无意义地沉迷在屏幕里。

### 1.2 我们是什么

LifeUp 是一个**用游戏化外壳承载真实人生成长的收集养成类数据化工具**。

核心使命只有一句话：

> **让玩家通过真实地投入时间学习技能，感受到自己在现实生活中变得越来越强大、越来越幸福。**

### 1.3 核心用户价值

| 用户痛点 | LifeUp 的解法 |
|---------|--------------|
| 不知道时间花哪了 | 柳比歇夫式精确时间记录 + 日/周/月/年统计报告 |
| 学了东西没成果感 | 技能等级可视化 + 累计投入时间档案 + 图鉴收集 |
| 难以坚持 | streak（连续天数）+ 成就徽章 + 微习惯引导 |
| 看不到技能之间的联系 | 技能组合系统，发现复合能力 |
| 焦虑、觉得没进步 | 每天2-3小时即可看到数据上的成长，降低心理门槛 |

### 1.4 玩家心理模型：收集 + 养成 + 生产

```
现实中花时间学习 → 游戏里技能升级 → 获得能力标签与道具 → 用于实际生产/生活
     ↑                                                              ↓
     └──────────── 看到成果后更有动力投入 ──────────────────────────┘
```

**关键认知：** 技能的价值最终要落回现实。游戏里的等级只是**现实世界能力的镜像和量化**。

### 1.5 技能示例与组合逻辑

**单一技能的现实映射：**
- 编程 → 找到开发工作、做副业、自动化生活
- 英语 → 看原版资料、出国交流、外企机会
- 烹饪 → 健康饮食、生活幸福感、社交货币

**组合技能（Combo Skills）：**
用户可自定义组合规则，当两个相关技能均达到 LV2 及以上时，可激活组合技能：
- 英语 LV2 + 恋爱沟通 LV2 → 解锁「跨文化社交」
- 编程 LV2 + 英语 LV2 → 解锁「远程工作/出海开发」
- 烹饪 LV2 + 摄影 LV2 → 解锁「美食自媒体」

## 二、技术架构方案

### 2.1 技术栈：Kotlin + Jetpack Compose（纯原生 Android）

**选型决策：纯原生 Android 开发**

| 维度 | 选择理由 |
|-----|---------|
| 原生体验 | 100% Android 原生渲染，无 WebView 套壳，流畅度最高 |
| 性能 | 直接调用系统 API，启动速度和运行效率最优 |
| 体积 | APK 体积小，无框架运行时开销 |
| Material You | 完美支持 Android 12+ 动态取色和 Material 3 设计语言 |
| 生态 | Google 官方推荐方案，文档和社区支持最完善 |
| 长期维护 | Kotlin 是 Android 官方语言，Jetpack Compose 是官方 UI 框架 |

### 2.2 核心技术组件

| 组件 | 用途 | 版本 |
|-----|------|------|
| **Kotlin** | 主开发语言 | 2.0+ |
| **Jetpack Compose** | 声明式 UI 框架 | BOM 2024+ |
| **Material 3** | 设计系统 + 动态取色 | 1.3+ |
| **Room** | 本地 SQLite ORM | 2.6+ |
| **Hilt** | 依赖注入 | 2.51+ |
| **Kotlin Coroutines + Flow** | 异步与响应式数据流 | 1.8+ |
| **Navigation Compose** | 页面路由 | 2.7+ |
| **DataStore** | 轻量键值存储（设置项） | 1.1+ |
| **WorkManager** | 后台定时任务（每日备份） | 2.9+ |
| **Accompanist** | 系统UI控制器（状态栏等） | 0.34+ |

## 三、数据存储与云端备份架构

### 3.1 核心原则

> **所有数据以本地为绝对中心。云端唯一的作用是「异地备份」，不存在本地与云端的冲突概念。应用完全支持离线使用，所有核心功能在无网络环境下均可正常运行。**

### 3.2 本地存储方案

- **主数据库**：Room（SQLite ORM），Android 原生方案
- **设置项**：DataStore（键值对），替代 SharedPreferences
- **本地备份目录**：应用私有目录下的 `backups/` 文件夹
- **备份策略**：保留最近 30 天的本地每日快照
- **手动本地导出**：支持将完整数据导出为 JSON 文件至用户指定目录，方便本地备份和迁移

### 3.3 云端备份方案：双方案可切换

#### 方案 A：S3 兼容存储（七牛云 / 阿里云 OSS / AWS S3）

| 项目 | 说明 |
|-----|------|
| 存储内容 | AES-256 加密后的完整数据快照 |
| 认证方式 | AccessKey + SecretKey + Bucket + Endpoint |
| 文件命名 | `lifeup_backup_<device_id>_<yyyy-MM-dd>.enc` |
| 优先推荐 | 七牛云 / 阿里云 OSS（国内访问速度最优） |

#### 方案 B：WebDAV（坚果云 / 群晖 / Nextcloud）

| 项目 | 说明 |
|-----|------|
| 存储内容 | 同上 |
| 认证方式 | URL + 用户名 + 密码 |
| 文件路径 | `/LifeUp/backups/lifeup_backup_<device_id>_<yyyy-MM-dd>.enc` |

### 3.4 云端备份策略：按天保留，自动轮转

- 每日应用启动时自动检测并推送备份
- 云端始终保留 **最近 10 天** 的备份
- App 永远不会主动从云端拉取数据
- 用户可手动推送或手动拉取恢复

### 3.5 数据恢复流程

设置 → 数据恢复 → 展示云端10个历史备份 → 选择 → 预览摘要 → 确认覆盖（恢复前自动保存本地快照）

### 3.6 数据导出格式

- **结构化数据**：JSON 格式导出（技能、时间记录、待办等）
- **统计报告**：CSV 格式导出（周/月/年报数据）
- **分享卡片**：PNG 图片格式（技能成就卡、streak 纪念卡等）

### 3.7 隐私政策

- 所有数据默认仅存储于用户本地设备
- 云端备份内容均经 AES-256 加密，服务端无法读取明文
- 不收集任何用户行为分析数据
- 不要求任何非必要权限
- 用户可随时导出全部数据并删除账户信息

## 四、技能系统核心设计

### 4.1 等级体系：五阶成长

| 阶位 | 标识 | 累计时间门槛 | Streak 要求 | 每天约2h预计 | 卡片视觉 |
|:---:|:----:|:----------:|:----------:|:----------:|---------|
| 一阶 | LV1 | 0h | 无 | 立即 | 默认样式 |
| 二阶 | LV2 | 20h | 无 | 约2周 | 铜色边框 |
| 三阶 | LV3 | 200h | 无 | 约3个月 | 银色边框 |
| 四阶 | LV4 | 500h | 无 | 约8个月 | 金色边框+微光 |
| 五阶 | LV5 | 1000h | 无 | 约1.5年 | 传说级特效+C位置顶 |
| 满级 | LV5 MAX | 2000h | 无 | 约3年 | 传说满级专属徽标 |

**等级标识统一使用 LV1 ~ LV5，不使用中文称号。**

**阶位门槛可自定义：** 用户可为每个技能单独设置升级所需累计时间，覆盖默认值。Streak 不再作为升级的硬性要求，但保持 Streak 可获得经验加成和成就徽章。

### 4.2 技能分类体系

| 大类 | 主题色 | 对应属性 | 示例 |
|-----|--------|---------|------|
| 谋生技能 | 科技青 | 智力 | 编程、设计、翻译 |
| 社交技能 | 暖橙 | 魅力 | 恋爱沟通、幽默表达 |
| 语言技能 | 宝石蓝 | 感知 | 英语、日语 |
| 生活技能 | 草木绿 | 灵巧 | 烹饪、理财 |
| 身体技能 | 活力红 | 力量 | 跑步、力量训练 |
| 心智技能 | 紫罗兰 | 意志 | 批判性思维、情绪管理 |
| 艺术技能 | 玫瑰金 | 创造 | 绘画、摄影、音乐 |

> 创建技能时需从九大属性维度中选择一个绑定，上述为默认推荐映射，用户可自由覆盖。

### 4.3 技能组合（Combo）系统

**开放自定义组合：** 用户可自由创建组合规则，不再限制为预设固定组合。

每条组合规则包含：
- 规则名称（如「跨文化社交」）
- 主技能 + 副技能（从已有技能中选择）
- 阶位要求（默认双方均需达到 LV2）
- 经验加成（默认 1.10x）
- 现实建议（文字描述，如「可以尝试跨国远程工作」）
- 连接线颜色（自定义）

**触发条件：** 当两个技能同时满足设定的阶位要求时自动激活，提供经验加成和现实行动建议。任意两个达到 LV2 的技能均可自由组合。

### 4.4 技能图鉴展览馆

采用图鉴收集风格，强化收集成就感：

```
技能图鉴
    ├── 收藏夹（用户置顶的常用技能）
    │       ├── 编程 · LV3
    │       └── 英语 · LV2
    ├── 展厅「2025年度目标」（用户自定义）
    │       ├── 编程 · LV3
    │       └── 英语 · LV2
    ├── 展厅「谋生技能馆」
    │       ├── 编程
    │       └── 设计
    ├── 成就墙
    │       ├── 已解锁 12 个技能
    │       ├── LV3+ 技能 3 个
    │       └── LV5 传说技能 0 个
    └── 未归档技能
```

**图鉴特性：**
- 每个技能有专属展示卡片，展示等级徽章和分类色块标记
- 点击卡片可查看技能成长曲线和各等级解锁徽章
- 支持一键分享技能卡片到社交平台
- 新解锁技能显示"NEW"角标
- 每个展厅有自定义名称、主题色、图标，内部可拖拽排序，有封面技能（C位）
- 成就墙展示解锁技能数量、高等级技能数量等里程碑

### 4.5 技能图标系统

| 层级 | 名称 | 解锁条件 | 风格 |
|-----|------|---------|------|
| 基础库 | 轮廓图标 | 默认可用 | 2px 线框风格 |
| 进阶库 | 面性图标 | LV2 解锁 | 单色填充+微渐变 |
| 传说库 | 动态徽标 | LV5 解锁 | 带微动效 |

支持用户上传自定义 SVG（LV2 即可解锁自定义图标上传，不再限制为 LV3）。

## 五、应用导航与页面架构

### 5.1 底部 Tab 栏：四大主入口

| Tab | 名称 | 核心定位 |
|:---:|------|---------|
| ① | **今日** | 每日待办规划与能量获取 |
| ② | **技能** | 技能图鉴展览、技能详情、计时 |
| ③ | **角色馆** | 角色换装、道具收集、属性面板 |
| ④ | **我的** | 个人档案、统计、设置、数据同步 |

### 5.2 Tab ①：今日

#### 5.2.1 页面结构

首页分为两个区域，每日总上限 10 项：

| 区域 | 上限 | 说明 |
|-----|------|------|
| 预设微习惯区 | 最多 5 项 | 无压力轻量化小事（如喝杯水、伸懒腰、早睡），引导用户先行动起来 |
| 自定义待办区 | 最多 5 项 | 用户自行创建，无需绑定技能 |

#### 5.2.2 待办类型与优先级

- **事项类型**：普通待办 + 技能关联待办
- **优先级标签**：红（高）/ 黄（中）/ 绿（低），用颜色区分任务优先级
- **默认排序规则**：高优先级 → 中优先级 → 低优先级 → 未设置优先级

#### 5.2.3 能量值机制

| 行为 | 能量变化 | 限制 |
|-----|---------|------|
| 完成普通待办 1 项 | +1 点 | 每天最多 10 点 |
| 完成高优先级待办 1 项 | +1.5 点 | 计入每日 10 点上限 |
| 创建新技能 | -2 点 | — |
| 为技能增加熟练度 | -1 点 | — |

- 每日 00:00 能量重置为 0，不累积
- 首页顶部和技能页顶部均显示实时能量值

#### 5.2.4 Streak 连续完成机制

- 连续完成天数 streak：每日至少完成 1 项待办即计为 1 天
- 中断后 streak 重置为 0，但保留历史最高 streak 记录
- 分级成就徽章：

| 累计连续天数 | 解锁徽章 |
|:----------:|---------|
| 7 天 | 铜色坚持徽章 |
| 30 天 | 银色毅力徽章 |
| 100 天 | 金色自律徽章 |
| 365 天 | 传说级传奇徽章 |

- 历史最高 streak 可生成分享纪念卡片

#### 5.2.5 快捷入口

- 手动补录时间
- 自由计时
- 启动技能计时（消耗 1 点能量）

### 5.3 Tab ②：技能

#### 5.3.1 技能图鉴列表页

- 以卡片形式展示所有技能
- 每张卡片展示：等级标识（LV1-LV5）、累计时长、升级进度条、分类色块标记、等级徽章
- 支持拖拽排序
- 顶部显示实时能量值

#### 5.3.2 技能详情页

- 累计时长统计
- 升级进度条（距离下一等级的进度）
- 历史计时记录：按年份一级折叠 → 展开后按月展示，单条记录仅显示月/日/时/分
- 自定义修改等级门槛
- 快速添加时长按钮
- 最近 7 天使用频次图表
- 技能成长曲线
- 各等级解锁徽章展示
- 一键分享技能卡片

#### 5.3.3 技能创建页

- 常用分类快捷选择栏
- 高频技能模板（如"编程"、"英语"、"跑步"等，一键创建并自动填充推荐配置）
- 创建技能需消耗 2 点能量
- 需选择绑定的属性维度（从九大属性中选择）
- 可选择绑定道具模板

### 5.4 Tab ③：角色馆

#### 5.4.1 核心定位

角色馆是游戏化反馈中心，以角色换装和道具收集为核心，让用户在提升技能的同时获得可视化的角色成长反馈。

#### 5.4.2 角色形象展示

- 角色形象随穿搭道具实时变化
- 支持保存多套搭配方案，一键切换
- 角色状态面板展示九大属性值和当前等级

#### 5.4.3 道具系统

**道具获取：** 用户学习升级技能时，解锁对应技能的专属道具。

**道具分级：**

| 道具等级 | 对应技能等级 | 外观风格 |
|:-------:|:----------:|---------|
| 普通 | LV1 | 基础轮廓，朴素设计 |
| 精良 | LV2 | 铜色边框，细节丰富 |
| 稀有 | LV3 | 银色边框，光效点缀 |
| 史诗 | LV4 | 金色边框，华丽光效 |
| 传说 | LV5 | 传说级特效，独特动效 |

每个道具有独立的文字描述和属性加成值。

**道具模板：** 预设 100 种通用道具模板，用户创建技能时可选择绑定对应道具，也可自定义道具名称和外观。

**背包系统：** 所有已解锁道具存放在背包中，支持按类型/等级/获取时间筛选。

**换装系统：** 用户可自由给角色穿搭换装，道具穿戴后对角色产生对应属性加成。

#### 5.4.4 九大属性维度

| 属性 | 说明 | 典型关联技能 |
|-----|------|------------|
| 力量 | 身体素质与体能 | 跑步、力量训练、武术 |
| 智力 | 逻辑思维与专业技能 | 编程、数学、研究 |
| 魅力 | 社交影响与表达 | 演讲、恋爱沟通、领导力 |
| 感知 | 语言理解与观察力 | 英语、日语、阅读 |
| 创造 | 艺术想象与创作 | 绘画、摄影、音乐 |
| 意志 | 自律与情绪管理 | 冥想、情绪管理、批判性思维 |
| 灵巧 | 生活技能与动手能力 | 烹饪、理财、手工 |
| 耐力 | 持续性与习惯坚持 | 习惯养成、长期项目 |
| 幸运 | 可配置属性 | 用户自定义 |

**属性值计算：** 创建技能时需绑定一个属性维度，按技能等级阶梯增加属性值：

| 技能等级 | 属性加成 |
|:-------:|:-------:|
| LV1 | +1 点 |
| LV2 | +3 点 |
| LV3 | +6 点 |
| LV4 | +10 点 |
| LV5 | +15 点 |

**幸运属性：** 可由用户自定义配置基础值，保留少量随机波动（±2），增加趣味性。

#### 5.4.5 成就墙

- 展示已解锁成就：技能里程碑、streak 纪录、道具收集进度等
- 成就解锁时弹出庆祝动效

## 六、时间记录与统计系统

### 6.1 柳比歇夫时间记录法

#### 6.1.1 两种记录方式

- **实时计时**：点击开始，实时记录，计时器前台服务 + 通知栏常驻
- **手动补录**：输入起始和截止时间（精确到分钟），系统自动计算时长

单条记录不超过 12 小时，重叠时间段提示确认。

#### 6.1.2 专注标记

每次计时结束后，用户可手动标记本次计时为：
- **有效专注**：真正投入学习的时长
- **非专注**：分心或效率低下的时长

此标记为可选操作，不影响基础计时记录，但影响统计报告中的专注率分析。

#### 6.1.3 柳比歇夫复盘页

- 按时间线展示每日各技能的投入记录
- 支持按日期搜索筛选
- 所有记录均绑定对应技能
- 区分显示有效专注时长和非专注时长
- 满足精细化复盘需求

### 6.2 技能统计报告

- **预设快捷查看**：周报 / 月报 / 年报
- **自定义时间段**：支持选择任意起止日期
- **报告内容**：
  - 对应时段内各技能的时长投入
  - 投入时长前三的技能
  - 成长变化对比
  - 有效专注率

### 6.3 时间记录展示规则

- 按年份做一级折叠分组
- 展开后按月展示
- 单条记录仅显示月/日/时/分，不保留冗余信息

## 七、数值策划体系

### 7.1 核心数值哲学

> **时间是最稀缺的资源。数值系统的使命是让每一分钟都被看见、被肯定、被积累。**

### 7.2 核心资源三角：能量 · 经验 · 金币

| 资源 | 获取方式 | 主要消耗 | 设计意图 |
|-----|---------|---------|---------|
| **能量** | 完成今日待办 | 创建技能、增加熟练度 | 限制操作频率，鼓励先规划再行动 |
| **经验** | 投资性计时 | 无消耗，自动累积 | 量化时间投入的成长价值 |
| **金币** | 所有计时 | 购买道具、解锁外观 | 让所有时间都有经济回报 |

### 7.3 能量值系统

| 行为 | 能量变化 | 限制 |
|-----|---------|------|
| 完成普通待办 1 项 | +1 点 | 每天最多 10 点 |
| 完成高优先级待办 1 项 | +1.5 点 | 计入每日上限 |
| 创建新技能 | -2 点 | — |
| 为技能增加熟练度 | -1 点 | — |

每日 00:00 重置为 0，不累积。

### 7.4 经验值获取体系（乘区制）

```
基础经验 = 投入分钟数 × 1 EXP/分钟

总经验 = 基础经验 × 阶位乘区 × 装备乘区 × 组合乘区 × Streak乘区 × 首击乘区 × 衰减系数
```

| 乘区 | 计算方式 |
|-----|---------|
| 阶位乘区 | LV1 1.0x / LV2 1.12x / LV3 1.28x / LV4 1.48x / LV5 1.75x |
| 装备乘区 | 所有装备效果加算后转换 |
| 组合乘区 | 激活组合中最高加成，默认1.10x |
| Streak乘区 | 1 + min(连续天数,30) × 0.005，上限1.15x |
| 首击乘区 | 每日首次投资性计时 1.20x |
| 衰减系数 | 见下方 Soft Cap |

### 7.5 每日收益 Soft Cap（简化版）

| 当日累计投入 | 衰减系数 |
|------------|---------|
| 0 - 120 分钟 | 100% |
| 120 - 240 分钟 | 75% |
| 240 - 360 分钟 | 50% |
| 360 分钟+ | 25% |

> 相比原方案减少了衰减梯度，降低用户心理压力，鼓励持续投入。

### 7.6 等级晋升数值

| 等级 | 累计时间 | Streak 要求 | 每天2h预计 |
|:---:|:-------:|:----------:|:----------:|
| LV1 | 0h | 无 | 立即 |
| LV2 | 20h | 无 | 约2周 |
| LV3 | 200h | 无 | 约3个月 |
| LV4 | 500h | 无 | 约8个月 |
| LV5 | 1000h | 无 | 约1.5年 |
| LV5 MAX | 2000h | 无 | 约3年 |

> 所有等级门槛均可由用户自定义调整。

### 7.7 金币经济系统

**获取：**
| 活动类型 | 金币/分钟 |
|---------|----------|
| 投资性计时 | 1 金币 |
| 消耗性计时 | 0.2 金币 |
| 每日首击奖励 | +50 金币 |
| 等级晋升 | LV2 +500 / LV3 +2000 / LV4 +5000 / LV5 +10000 |

**用途：** 购买道具(500-5000G) / 解锁图标库 / 角色外观 / 重置技能

### 7.8 健康与反内卷设计

| 机制 | 说明 |
|-----|------|
| 平滑收益衰减 | 同一技能当日超1小时后收益平滑下降 |
| 健康提醒 | 连续计时超90分钟弹出休息提醒 |
| Streak 睡眠保护 | 23:00-06:00 计时不计入 streak |
| 消耗性时间温和记录 | 刷手机也有微量回报，不制造焦虑 |
| 每周休息日提示 | 连续7天有记录时提示可休息 |
| 断 streak 不惩罚 | 保持 streak 有加成，中断不扣减已有成果 |

## 八、视觉与交互设计

### 8.1 视觉定位

> **精致的数据化人生仪表盘**

- **风格**：Material 3 + 游戏化数据可视化
- **色彩**：**默认深色模式**，技能卡片用自定义色点缀，支持 Android 12+ 动态取色
- **动效**：升级庆祝动效，日常界面克制
- **字体**：系统默认 + 等宽数字字体展示时间数据
- **卡片圆角**：增大技能卡片圆角半径，营造柔和精致的视觉感受

### 8.2 交互原则

| 原则 | 实践 |
|-----|------|
| 即时反馈 | 计时结束1秒内弹出结算面板 |
| 损失厌恶转增益 | 断 streak 不惩罚，保持 streak 有加成 |
| 进度可视化 | 所有技能有明确进度条 |
| 收集满足感 | 图鉴展览模式，新技能有"NEW"角标 |
| 默认不做决定 | 计时器默认关联上次技能 |
| 微习惯引导 | 首页预设无压力小事，降低行动门槛 |

## 十一、总结

LifeUp 的终极愿景是成为用户的**"第二大脑"和"人生操作系统"**。

它不替代真实生活，而是**放大真实生活的意义**。当用户在现实中花2小时学英语，打开 LifeUp 看到自己的「英语」技能从 LV1 走向 LV2，看到「累计投入127小时」的档案，看到「跨文化社交」的组合技能被点亮，看到角色馆里新解锁的道具为角色带来属性提升——这种**确定性、可量化、可积累、可收集**的感觉，就是我们想给用户的全部。

**时间没有浪费。每一分钟都在让你升级。**
